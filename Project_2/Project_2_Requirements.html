CS 158A                                  Introduction to Computer Networks             Spring 2015

 

Project II – TCP and UDP Performance Measurement

Last Update: 3/3/2015

 

1.      Group

Form a group of 2-4 members and elect a coordinator.

 

2.      Objective

1) To use sockets for communication between machines, 2) to learn about performance metrics such as round-trip time (latency) and throughput in computer networks, 3) to measure the performance of TCP and UDP between pairs of networked computers.

 

3.      Project Description

Sockets are inter-process communication mechanisms that allow processes to communicate with each other, even if they are on different machines.  In this project, you will use Internet Sockets to send different-sized messages among several machines and measure latency and throughput. You may also use Sockets to simulate CSMA/CD in a simple LAN setting.

 

4.      Procedure

a.       Familiarize yourself with sockets. Note: Receive 10% extra bonus when you use C or C++ (instead of Java) for Project III. 

b.      UDP only

i.                    Write client and server socket programs, using UDP as the transport protocol, to send a one-byte message from one machine (client) to another machine (server), and have the other machine (server) return the message.

ii.                  Repeat the above process in a loop 1,000 times, and calculate the average round-trip time (latency) for a one-byte message. Average RTT is given by [total latency to loop 1000 times divided by 1000].

iii.                Increase the message size in the above program to 1Kbyte, 4KB, 8 KB, 16KB, 32 KB, and 64KB (if your system does not support 64*1024 bytes, use 64,000 bytes). For each, loop 100 times. Discuss any problems encountered in this process, and measure the average round-trip time (delay) for each case.  Determine the effective throughput (size/delay, in bits/sec) for each message size. Determine the number of lost packets in UDP; lost packets may still be included to calculate RTT, but not effective throughput.

iv.                Plot (i) round-trip time vs. message size, (ii) throughput vs. message size, and (iii) number of lost packets vs. message size.  Discuss the results. (Hint: You may use traceroute or any other way to figure out the network path between the server and the client and explain the performance.)  Describe the network(s) that the server and client are on, and the network path between server and client; again you may use traceroute to aid your description. Give the IP addresses of server and client if possible.

v.                  Repeat steps (i) – (iv) for two different clients (on two different machines) connecting to the same server; i.e., connecting to the same server application on the server machine. The two clients should concurrently send requests to the server; it is preferred that the two different clients be on two different LANs or subnets. Plot the results of the two clients on the same figure.

c.       TCP only - Repeat Step (b) using TCP as the transport protocol (you do not need to plot lost packets).

d.      TCP and UDP together – Repeat the experiments of TCP and UDP concurrently, at the same time, over the same network path (i.e., repeat Steps (b) and (c) concurrently). Plot the results on the same figure. Discuss the results.

e.       TCP only: Investigate the different options and parameters that you can set for a TCP connection. Experiments with at least two different options to see how they affect TCP performance. Plot the results of round-trip time for different options/parameters on the same figure to support your finding. Repeat for the results of throughput.
